# 高校校历管理系统设计方案

## 1. 项目概述
构建一个面向高校教务处的校历管理系统，实现对每学年的**春秋学期、寒暑假、法定节假日、统一安排活动**的全生命周期数字化管理。系统需要提供可视化的校历编排、冲突检测、周次自动计算、发布与导出等核心能力，以替代传统 Excel 手工维护模式，提升教务管理效率与规范性。

**用户角色：**
- **教务处管理员**：负责学年初始化、学期设定、假期与节假日维护、活动安排、校历审核发布。
- **师生（查看者）**：可通过门户页面浏览已发布校历，按学期、月、周查看，订阅日历（iCal）。

## 2. 需求分析
### 2.1 功能性需求
- **学年学期管理**：创建学年（如“2026-2027学年”），设定秋季学期、春季学期起止日期；支持学期中包含的教学周数自动计算。
- **寒暑假管理**：定义寒假、暑假的起止日期，关联到对应学年；支持跨年寒假（如2027年1月-2月）归属处理。
- **法定节假日与调休**：内置国务院发布的法定节假日模板，支持调休工作日标记；可自定义学校特有放假日（如校庆日）。
- **统一活动管理**：添加校运会、新生报到、考试周、毕业典礼等全校性活动，设定日期范围或具体日期，可关联学期与周次。
- **校历视图与展示**：提供月视图、学期视图、列表视图；清晰展示周次、假期、活动标注，支持颜色区分类型。
- **冲突检测**：新增活动时自动检测是否与已锁定假期、法定节假日或其他重要活动重叠，给出警告。
- **导出与订阅**：支持导出为 Excel、高清图片；生成 iCal 订阅链接，供师生同步到个人日历。
- **版本与发布**：每学年校历可保存为草稿，发布后锁定，修改时生成新版本并记录变更日志。

### 2.2 非功能性需求
- 浏览器端响应式设计，移动端友好查看。
- 数据一致性，事务保障学期日期与周次计算。
- 权限控制：仅管理员可编辑，师生只读。
- 可维护性：业务规则（如学期周数、假期类型）可配置。

## 3. 功能模块设计
### 3.1 学年与学期管理
- 新增学年，自动生成默认秋季、春季学期草稿（起止日期可调）。
- 学期支持设置教学周起始日（如周一），自动推算周次列表。
- 特殊学期处理：如“夏季学期”可选扩展。

### 3.2 假期管理
- **假期类型**：寒假、暑假、法定节假日、调休工作日、学校自定义假日。
- 假期可设定日期范围，支持跨月跨年。
- 法定节假日预设库：系统内置近五年国务院放假安排，可一键导入并允许微调。
- 假期与学期关联：寒假自动关联秋季学期结束后的区间，暑假关联春季学期结束后。

### 3.3 节假日与调休
- 独立管理“节假日日历”，标记某日为“节假日”、“调休上班”或“普通工作日”。
- 支持批量导入（Excel）或通过预设模板生成。

### 3.4 统一活动管理
- 活动属性：名称、类型（教学/考试/文体/行政）、日期范围、关联学期、优先级、是否全校停课。
- 重复活动支持：如每年同一周次举办，可设为年度重复规则。
- 活动在日历视图上高亮显示，悬停展示详情。

### 3.5 校历可视化与查询
- **学期视图**：以学期为单位，横向展示周次条，标注假期区间和活动点。
- **月/周视图**：传统日历风格，颜色标记不同类型事件。
- **列表视图**：按时间排序所有事件，支持筛选类型。

### 3.6 系统管理与配置
- 学期默认规则（秋季9月到1月，春季2月到7月，可调整）。
- 节假日库更新：管理员可维护远期模板。
- 操作日志与校历版本对比。

## 5. 业务流程设计
### 5.1 学年校历编排流程
1. **创建新学年**：管理员选定年份，系统自动生成学年记录和春秋学期默认起止（参考配置）。
2. **设定学期精确日期**：微调学期起止，系统计算教学周数并生成周次序列。
3. **导入法定节假日**：选择对应年度模板，一键生成当年节假日及调休日。
4. **定义寒暑假**：根据学期结束和开始日期，拖拽设定寒假、暑假区间。
5. **添加统一活动**：在日历界面点击日期或周次添加活动，填写信息后保存，系统执行冲突检查。
6. **预览与调整**：通过学期视图查看整体校历，检查假期与活动布局。
7. **发布校历**：确认无误后发布，系统生成快照，状态变为已发布，前端展示更新。
8. **后续修改**：若需调整，创建新版本，重新编辑后发布，旧版本归档可查。

### 5.2 冲突检测规则
- 活动日期不能落在已发布的**法定节假日**（放假）或**寒暑假**内（可配置警告级别）。
- 多个活动在同日重叠时，根据优先级提醒。
- 调休工作日允许安排活动，但须标注“补课日”等。
- 检测结果以弹窗列表呈现，允许管理员强制忽略并记录原因。

## 6. 系统架构建议
- **架构风格**：B/S 前后端分离
- **前端**：Vue3 + Element Plus / React + Ant Design，使用 FullCalendar 组件库进行日历渲染。
- **后端**：Spring Boot (Java) 或 Django REST Framework (Python)，提供 RESTful API。
- **数据库**：MySQL 8.0 / PostgreSQL，利用 JSON 字段存储快照。
- **部署**：Nginx + 应用服务器，支持容器化（Docker）。

**核心 API 设计举例：**
- `POST /api/semester` - 创建学期
- `GET /api/calendar?yearId=1&view=month` - 获取校历事件数据（聚合学期、假期、活动）
- `POST /api/events` - 新增活动，返回冲突检测结果
- `POST /api/calendar/publish` - 发布校历

## 7. 关键技术实现
### 7.1 教学周自动计算
- 学期开始日期若为周一，则第一周从该日开始；否则第一周包含开始日期所在周的周一。
- 周次计算：`weekNumber = floor((date - firstMonday) / 7) + 1`
- 后端提供工具类，任何日期可快速转换周次和学期。

### 7.2 校历视图数据聚合
- 视图 API 将学期、假期、活动数据合并为统一的事件格式：
  ```json
  {
    "title": "开学典礼",
    "start": "2026-09-01",
    "end": "2026-09-01",
    "color": "#1890ff",
    "type": "event",
    "extendedProps": { "week": 1, "semester": "秋季学期" }
  }
  ```
- 假期区间渲染为背景色条，在 FullCalendar 中使用 `background event` 或插件实现。

### 7.3 iCal 订阅生成
- 为已发布学年生成 `.ics` 文件静态链接，包含学期起止、假期、活动。师生可通过 URL 订阅，自动同步。
- 使用类库（如 `ical-generator` Node.js 或 Java `ical4j`）动态生成。

### 7.4 版本快照
- 发布时，将学期、假期、活动数据序列化为 JSON 存入 `snapshot_data`，用于历史版本回看与对比，防止后续编辑影响已发布内容。

## 8. 界面原型思路
- **后台管理**：左侧菜单（学年管理、学期设置、假期设置、活动安排、校历发布），主工作区采用日历控件和表单弹窗。
- **前台展示**：顶部学年切换，核心区域学期时间线 + 月历，点击日期显示详情。移动端可上下滑动查看连续月份。

## 9. 扩展性设计
- 可对接教务系统，将课表、考试安排自动转化为活动展示。
- 支持多校区（不同校区可能有不同活动日期），增加校区维度。
- 校历数据可对外提供 OpenAPI，供校园 App 调用。

本设计方案覆盖了高校校历管理的全部核心场景，具备从基础数据维护到可视化发布、导出的完整能力，可按此框架进行技术选型与迭代开发。如需进一步细化某一模块（如详细 API 定义、前端组件拆分等），可以继续深入补充。
